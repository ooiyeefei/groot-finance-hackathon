# Implementation Plan: Improve AI Action Center Insights Quality

**Branch**: `001-improve-action-center` | **Date**: 2026-03-14 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-improve-action-center/spec.md`

## Summary

Overhaul the AI Action Center detection pipeline to produce CFO-grade insights by: (1) resolving category IDs to human-readable names, (2) separating AP supplier analysis from expense-claim merchant analysis, (3) adding materiality-based priority scoring, (4) consolidating pattern-level insights into summary cards with keyword-overlap dedup, (5) improving LLM prompts with domain-separation rules, (6) adding Ask AI prepopulated prompts with suggestion chips, and (7) one-time cleanup + re-run of existing insights.

## Implementation Approach

### Phase A: Backend Detection Pipeline (P1 — actionCenterJobs.ts)

**A1. Category Name Resolution**
- Add `resolveCategoryName()` helper that queries `business_expense_categories` for the business, maps category ID → `category_name`. Fallback: strip `_[a-z0-9]+$` suffix, capitalize first letter.
- Call it in: `runAnomalyDetection` (line ~500), `analyzeNewTransaction` (line ~1557), `getBusinessSummary` (line ~137), and LLM prompt construction.

**A2. Domain-Aware Entry Filtering**
- Add `classifyEntryDomain()` helper: returns `"ap_vendor"` if entry has `vendorId` (linked to `vendors` table), `"expense_claim"` if no `vendorId`, or `"cogs"` if `transactionType === "Cost of Goods Sold"`.
- Modify `runVendorConcentration` (line ~705): filter to only `ap_vendor` and `cogs` entries.
- Modify `runVendorSpendingChanges` (line ~812): filter to only `ap_vendor` entries.
- Modify `runVendorRiskAnalysis` (line ~917): already only queries `vendors` table — just fix terminology in output text.
- Fix all insight titles/descriptions: "vendor" → "supplier" for AP, "merchant" for expense claims.

**A3. Materiality-Based Priority Scoring**
- Add `computeMaterialityPriority()` that considers both σ-deviation AND amount as % of monthly expenses.
- Logic: if amount < 0.1% monthly expenses → suppress entirely. If amount < 1% → cap at "low". Otherwise use existing σ-based logic.
- Apply in `runAnomalyDetection` and `analyzeNewTransaction`.

**A4. Pattern-Level Insight Consolidation**
- Modify `runVendorConcentration`: instead of creating one insight per vendor per category, collect ALL concentration findings, then create ONE summary insight with all affected suppliers listed in description and metadata.
- Same for `runVendorSpendingChanges`: consolidate into one "Supplier Spending Changes" card.
- Keep transaction-level alerts (anomalies, duplicates, pricing surges) as individual cards.

**A5. Business Summary Domain Separation**
- Modify `getBusinessSummary` and `getBusinessSummaryByStringId`: split `vendorSpend` into two buckets — `supplierSpend` (entries with `vendorId`) and `merchantSpend` (entries without `vendorId`).
- Output: `topSuppliers` (AP) and `topMerchants` (Expense Claims) with clear labels.

**A6. LLM Prompt Improvements**
- Modify `enrichInsight` system prompt: add domain-separation rules, use "supplier"/"merchant" terminology.
- Modify `runAIDiscovery` system prompt: add rules about materiality, domain separation, forbid generic "vendor diversification" advice for expense merchants, and enforce summary-card format for pattern findings.

### Phase B: Backend Dedup + Migration (P1 — actionCenterInsights.ts)

**B1. Keyword Overlap Dedup for LLM Insights**
- Add `computeJaccardSimilarity(title1, title2)` utility: tokenize (lowercase, split on spaces/punctuation), compute Jaccard coefficient.
- In `internalCreate`: before inserting, if `category` is from LLM discovery (metadata.aiDiscovered), check Jaccard similarity against all existing insight titles for the business. Reject if >0.6.

**B2. One-Time Migration Script**
- Add `resetAndRegenerate` internalMutation: deletes all `actionCenterInsights` for a business.
- Add `migrateAllBusinesses` internalAction: iterates businesses, calls `resetAndRegenerate`, then triggers `runDetectionAlgorithms` + `enrichRecentInsights` for each.

### Phase C: Frontend Ask AI UX (P2 — InsightCard.tsx)

**C1. Prepopulated Editable Prompt**
- Change `handleAskAI`: instead of dispatching full context as `message`, dispatch as `draftMessage` (new event detail field).
- Chat widget receives `draftMessage` → sets it in the input field (not auto-sent).
- Prompt text: concise, human-readable (e.g., "Tell me more about: {insight.title}. What data supports this finding?").

**C2. Suggestion Chips**
- Add `getInsightQuestionChips(insight)` function that returns 2-3 context-aware questions based on `insight.category`.
- Dispatch chips alongside draftMessage in the event detail.
- Chat widget renders chips below input as clickable buttons.

### Phase D: Testing & Cleanup

**D1. Build Verification**
- `npm run build` must pass.

**D2. Pipeline Verification**
- Run `npx convex run functions/actionCenterJobs:runProactiveAnalysis` on dev and verify:
  - No raw category IDs in insight titles
  - No "vendor" in user-facing text (only "supplier"/"merchant")
  - Pattern-level insights are consolidated
  - Materiality thresholds applied

**D3. Migration Execution**
- Run `npx convex run functions/actionCenterInsights:migrateAllBusinesses` on prod after deploy.
