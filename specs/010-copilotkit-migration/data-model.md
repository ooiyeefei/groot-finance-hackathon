# Data Model: CopilotKit Agent Migration

**Feature**: 010-copilotkit-migration
**Date**: 2026-02-11

## Overview

The data model is **unchanged** by this migration. All entities continue to be stored in their existing locations (Convex, Qdrant, Mem0). This document confirms the existing model and notes how CopilotKit interacts with it.

## Entities

### Conversation (Convex: `conversations` table)

| Field | Type | Description |
|-------|------|-------------|
| userId | string | Clerk user ID (owner of the conversation) |
| businessId | string? | Business context for multi-tenant isolation |
| title | string | Auto-generated or user-defined conversation title |
| language | string | Conversation language (en, th, id) |
| isActive | boolean | Whether the conversation is active (not archived) |
| messageCount | number | Denormalized count of messages |
| lastMessageContent | string | Preview of the last message |
| lastMessageRole | "user" \| "assistant" | Role of the last message sender |
| lastMessageAt | number | Timestamp of the last message |
| contextDocumentId | Id? | Optional linked document context |
| contextTransactionId | Id? | Optional linked transaction context |
| updatedAt | number | Last update timestamp |

**CopilotKit interaction**: CopilotKit does not directly read/write this table. The bridge hook (`use-copilot-chat.ts`) syncs between CopilotKit's in-memory message state and Convex.

### Message (Convex: `messages` table)

| Field | Type | Description |
|-------|------|-------------|
| conversationId | string | Parent conversation ID |
| role | "user" \| "assistant" | Message sender |
| content | string | Message text (markdown with citation markers) |
| metadata | object? | Optional metadata including citations and agent state |
| metadata.citations | CitationData[]? | Array of citation references |
| metadata.agentState | object? | Snapshot of agent state at message time |

**CopilotKit interaction**: After each CopilotKit message exchange completes, the bridge hook persists both user and assistant messages to Convex with their metadata.

### Citation (embedded in Message metadata)

| Field | Type | Description |
|-------|------|-------------|
| id | string | Unique citation identifier |
| source_name | string | Name of the source document |
| country | string | Country of the regulatory document |
| section | string? | Section within the document |
| pdf_url | string? | URL to the source PDF |
| page_number | number? | Page number in the PDF |
| text_coordinates | object? | Bounding box coordinates {x1, y1, x2, y2} |
| content_snippet | string | Excerpt from the source document |
| confidence_score | number | Similarity/relevance score (0-1) |
| official_url | string? | Link to official government page |

**CopilotKit interaction**: Citations are generated by the LangGraph agent and returned in the response metadata. The custom message renderer parses citation markers and renders the CitationOverlay.

### UserContext (runtime-only, not persisted)

| Field | Type | Description |
|-------|------|-------------|
| userId | string | Clerk user ID |
| businessId | string | Business context |
| role | string | User role (owner, admin, manager, finance_admin, employee) |

**CopilotKit interaction**: Extracted from Clerk auth at the CopilotKit runtime endpoint and passed to the LangGraph agent on each invocation.

### AgentState (LangGraph runtime, ephemeral)

| Field | Type | Description |
|-------|------|-------------|
| messages | BaseMessage[] | LangChain message history |
| userContext | UserContext | Current user context |
| securityValidated | boolean | Whether security validation passed |
| currentIntent | UserIntent? | Analyzed user intent |
| needsClarification | boolean | Whether agent needs more info |
| clarificationQuestions | string[] | Pending clarification questions |
| currentPhase | string | Current agent phase |
| citations | CitationData[] | Accumulated citations |
| isTopicAllowed | boolean | Topic guardrail result |
| failureCount | number | Circuit breaker counter |
| lastFailedTool | string? | Last tool that failed |

**CopilotKit interaction**: If using `useCoAgent`, the agent state can be shared bidirectionally with the frontend. Initially, we use `useCopilotChat` (simpler), so agent state is internal to the runtime.

### Memory (Mem0 / Qdrant)

| Field | Type | Description |
|-------|------|-------------|
| app_id | string | Business ID (multi-tenant key) |
| user_id | string | Clerk user ID |
| content | string | Memory content |
| metadata | object | Memory metadata (type, timestamp, etc.) |

**CopilotKit interaction**: None. Memory is managed entirely within the LangGraph agent's pipeline.

## State Transitions

### Conversation Lifecycle
```
Created → Active → (Messages exchanged) → Archived/Deleted
```
No changes from current behavior. CopilotKit does not manage conversation lifecycle — Convex handles it.

### Agent Phase Flow
```
validation → intent_analysis → clarification → execution → completed
```
No changes. CopilotKit wraps the agent at the runtime level; phase transitions are internal.

## Data Migration

**No data migration required.** The Convex schema is unchanged. Existing conversations and messages will be read by the new CopilotKit UI through the same Convex queries.
