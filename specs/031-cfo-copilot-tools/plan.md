# Implementation Plan: CFO Copilot Tools

**Branch**: `031-cfo-copilot-tools` | **Date**: 2026-03-21 | **Spec**: [spec.md](./spec.md)

## Summary

Add three CFO copilot capabilities to the chat agent: (1) monthly cash flow forecasting with AR/AP awareness and risk alerts, (2) PDF board report generation with 6 financial sections, and (3) tax reference knowledge base for Malaysia/Singapore factual data. All tools follow MCP-first architecture.

## Technical Context

**Language/Version**: TypeScript 5.9.3 (Node.js 20 for Lambda)
**Primary Dependencies**: PDFKit (new — PDF generation in Lambda), existing: Convex, Qdrant, AWS S3
**Storage**: S3 `finanseal-bucket` (PDF storage), Qdrant Cloud (tax KB embeddings), Convex (transaction data)
**Testing**: Manual chat testing (UAT), `npm run build` verification
**Target Platform**: MCP Lambda (AWS), Next.js frontend (Vercel)
**Project Type**: Web application (Next.js + Convex + AWS Lambda)
**Performance Goals**: Forecast <10s, PDF generation <30s
**Constraints**: Free-tier Convex bandwidth limits (use actions, not queries), MCP-first tool architecture
**Scale/Scope**: Single-tenant per business, 1-12 month forecast horizon

## Implementation Phases

### Phase A: P1 — Cash Flow Forecasting (MCP + Action Card)

1. **Extend `forecast-cash-flow.ts`** — Add `forecast_months`, `granularity`, `include_known_ar_ap` params. When `granularity=monthly`, aggregate daily data into monthly buckets. Query `sales_invoices` for AR and `invoices` for AP to include known future cash flows.

2. **Update contracts** — Add `MonthlyForecastOutput`, `MonthlyBucket`, `ForecastRiskAlert` types to `mcp-tools.ts`.

3. **Build `forecast-card.tsx`** — New action card with monthly bar chart (income vs expenses), running balance line, risk alert badges. Register in action-cards index.

4. **Update agent prompts** — Add `forecast_card` documentation (type 11) in `prompts.ts` with data schema and example triggers.

5. **RBAC** — Add forecast tool access for manager role alongside existing finance_admin/owner.

### Phase B: P2 — PDF Board Report (MCP + PDF + S3)

1. **Add PDFKit dependency** — `npm install pdfkit` (lightweight, no DOM).

2. **Create `pdf-builder.ts`** — Utility that takes report data and generates PDF buffer using PDFKit. Sections: cover page, P&L, cash flow, AR aging, AP aging, top vendors, trends.

3. **Create `generate-report-pdf.ts`** — New MCP tool. Queries Convex for report data, calls pdf-builder, uploads to S3, returns signed URL.

4. **Register tool** — Add to `index.ts`, `handler.ts`, `mcp-tools.ts`.

5. **Build `report-download-card.tsx`** — Action card with download button, filename, period, sections list.

6. **Update agent prompts** — Add `report_download` documentation in `prompts.ts`.

7. **RBAC** — Same as forecast: owner/finance_admin/manager.

### Phase C: P3 — Tax Reference KB

1. **Create `upload-tax-kb.ts` script** — Node.js script that reads tax reference content (hardcoded JSON array), generates embeddings via Gemini, and upserts to Qdrant `regulatory_kb` collection with `category: 'tax_reference'` metadata.

2. **Update `RegulatoryKnowledgeTool`** — Add disclaimer footer to tax reference responses. Detect advisory questions and decline with professional referral.

3. **Update agent prompts** — Add guidance on tax reference vs. advisory boundary.

4. **Run upload script** — Execute to populate KB.

### Phase D: Verification

1. `npm run build` — Must pass
2. `npx cdk deploy` — MCP Lambda with new tools
3. `npx convex deploy --yes` — If any Convex changes
4. Chat UAT — Test all three features with test accounts
