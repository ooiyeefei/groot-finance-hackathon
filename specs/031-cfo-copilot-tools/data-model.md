# Data Model: CFO Copilot Tools

## No New Convex Tables Required

All three features use existing tables and external storage. No schema changes needed.

## Data Flows

### P1: Cash Flow Forecast (Monthly)

**Input**: `forecast_months` (1-12), `scenario` (conservative/moderate/optimistic)

**Data Sources** (existing Convex tables):
- `journal_entry_lines` — historical income/expense by month (via `financialIntelligence:getMcpAccountingEntries`)
- `sales_invoices` — unpaid AR (sent invoices with due dates)
- `invoices` — posted AP (vendor invoices with due dates, paymentStatus != 'paid')

**Output Shape** (MCP tool response):
```typescript
interface MonthlyForecastOutput {
  months: MonthlyBucket[]
  summary: {
    current_balance: number
    runway_months: number
    scenario_used: 'conservative' | 'moderate' | 'optimistic'
    risk_level: 'low' | 'medium' | 'high' | 'critical'
    total_known_ar: number   // sum of unpaid sent invoices
    total_known_ap: number   // sum of posted unpaid AP
  }
  risk_alerts: RiskAlert[]
  currency: string
}

interface MonthlyBucket {
  month: string              // "2026-04"
  projected_income: number
  projected_expenses: number
  known_ar_due: number       // AR maturing this month
  known_ap_due: number       // AP due this month
  net_balance: number        // running cumulative balance
  confidence: 'high' | 'medium' | 'low'
}

interface RiskAlert {
  type: 'low_runway' | 'negative_balance' | 'ar_concentration'
  severity: 'critical' | 'warning' | 'info'
  month: string
  message: string
  recommendation: string
}
```

### P2: PDF Board Report

**Input**: `report_type` ('board_report'), `date_range` ({ start, end }), `sections` (optional array)

**Data Sources** (existing Convex tables):
- `journal_entry_lines` — P&L, cash flow data
- `sales_invoices` — AR aging
- `invoices` — AP aging, top vendors
- Forecast data — reuses P1 forecast tool internally

**Output Shape** (MCP tool response):
```typescript
interface GenerateReportOutput {
  report_url: string         // CloudFront signed URL (7-day expiry)
  filename: string           // "Board-Report-Q1-2026.pdf"
  sections_included: string[]
  date_range: { start: string; end: string }
  generated_at: string
}
```

**PDF Sections** (generated by PDFKit):
1. Cover page (business name, report type, date range)
2. P&L Summary (revenue, COGS, gross profit, operating expenses, net income)
3. Cash Flow Overview (opening balance, inflows, outflows, closing balance)
4. AR Aging (current, 30d, 60d, 90d+, total)
5. AP Aging (current, 30d, 60d, 90d+, total)
6. Top Vendors (top 10 by spend in period)
7. Trend Charts (monthly revenue, expenses, net income for trailing 6 months)

**S3 Storage Path**: `reports/{businessId}/board-report/{timestamp}-Board-Report-{period}.pdf`

### P3: Tax Reference KB

**No new data model** — uses existing Qdrant `regulatory_kb` collection.

**New KB entries shape** (uploaded to Qdrant):
```typescript
interface TaxReferenceEntry {
  // Qdrant point payload
  text: string               // Factual content (rates, deadlines, thresholds)
  category: 'tax_reference'  // Distinguishes from regulatory/compliance content
  jurisdiction: 'MY' | 'SG'
  topic: 'corporate_tax' | 'gst' | 'filing_deadlines' | 'thresholds'
  effective_date: string     // When this fact became effective
  source: string             // e.g., "LHDN Guidelines 2026", "IRAS GST Guide"
  created_at: string
}
```

**Content to upload** (initial set):
- Malaysia corporate tax rates (SME vs standard, YA 2026)
- Malaysia filing deadlines (Form C, Form E, estimated tax)
- Singapore GST rates and registration thresholds
- Singapore GST filing calendar (quarterly/monthly)
- Common deductible expense categories per jurisdiction
