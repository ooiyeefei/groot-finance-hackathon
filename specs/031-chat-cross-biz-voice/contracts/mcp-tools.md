# MCP Tool Contracts: Issue #352

## Tool: `send_email_report`

**Purpose**: Send a formatted financial report via email to specified recipients.

### Input Schema

```typescript
{
  report_type: string       // "ap_aging" | "ar_aging" | "cash_flow" | "pnl" | "expense_summary"
  report_data: object       // Structured report data (generated by agent before calling this tool)
  recipients: string[]      // Email addresses (1-10)
  subject?: string          // Optional custom subject line
  confirmed: boolean        // Must be true to actually send (two-phase confirmation)
  period?: string           // Report period, e.g., "2026-03" or "2026-Q1"
}
```

### Output Schema

```typescript
// When confirmed: false (preview phase)
{
  preview: true
  confirmation_message: string  // "Ready to send AP Aging Report to john@company.com. Please confirm."
  recipients: string[]
  report_type: string
}

// When confirmed: true (send phase)
{
  success: boolean
  message_ids: string[]         // SES message IDs
  recipients_sent: string[]     // Successfully sent to
  recipients_failed: string[]   // Failed recipients (if any)
  daily_sends_remaining: number // Remaining sends for today (out of 50)
}
```

### RBAC

- **Allowed roles**: `finance_admin`, `owner`
- **Denied roles**: `employee`, `manager` → returns error with explanation

### Rate Limit

- 50 emails per business per day
- Counted by `email_send_logs` entries with matching businessId and sentAt within current UTC day

## Tool: `compare_to_industry`

**Purpose**: Compare a business's financial metrics against anonymized industry benchmarks.

### Input Schema

```typescript
{
  metric: string           // "gross_margin" | "cogs_ratio" | "opex_ratio" | "ar_days" | "ap_days"
  period?: string          // Optional, defaults to most recent quarter. Format: "2026-Q1" or "2026-03"
}
```

### Output Schema

```typescript
// When business is opted in and sufficient data exists
{
  success: true
  metric: string
  business_value: number        // The querying business's value for this metric
  industry_group: string        // e.g., "Wholesale Trade"
  percentile: number            // 0-100, where the business ranks
  industry_average: number
  industry_median: number
  p25: number
  p75: number
  sample_size: number           // Number of businesses in comparison
  period: string
  recommendations: string[]     // AI-generated suggestions based on ranking
}

// When business is NOT opted in
{
  success: false
  reason: "not_opted_in"
  message: string               // Explanation of benchmarking feature and how to opt in
}

// When insufficient peers
{
  success: false
  reason: "insufficient_data"
  message: string
  current_sample_size: number
  minimum_required: number      // 10
}
```

### RBAC

- **Opt-in/out toggle**: `finance_admin`, `owner` only
- **Read benchmarks**: All roles within an opted-in business

## Tool: `toggle_benchmarking`

**Purpose**: Opt a business in or out of anonymized benchmarking.

### Input Schema

```typescript
{
  action: "opt_in" | "opt_out"
}
```

### Output Schema

```typescript
{
  success: boolean
  is_active: boolean            // Current opt-in status after action
  industry_group: string        // The business's industry category
  message: string               // Confirmation message
}
```

### RBAC

- **Allowed roles**: `finance_admin`, `owner`
- **Denied roles**: `employee`, `manager` → returns error
