# Implementation Plan: Price History Tracking

**Branch**: `032-price-history-tracking` | **Date**: 2026-03-22 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/032-price-history-tracking/spec.md`

## Summary

Add selling price history tracking to complement the existing vendor intelligence (purchase price tracking). When a sales invoice is sent, the system automatically captures selling prices for catalog-linked line items. A new catalog item detail page at `/sales-invoices/catalog/[itemId]` provides tabs for Sales History, Purchase History, and Price Comparison with margin analysis. A vendor item mapping table bridges the gap between vendor price data and catalog items using fuzzy-match bootstrapping.

## Key Integration Points

1. **salesInvoices.send()** (line ~610): After `ctx.db.patch(args.id, { status: "sent" })`, call `ctx.scheduler.runAfter(0, internal.functions.sellingPriceHistory.recordFromSalesInvoice, {...})` for catalog-linked line items
2. **salesInvoices.voidInvoice()** (line ~700): Call `internal.functions.sellingPriceHistory.archiveBySalesInvoice` to soft-delete records
3. **vendorPriceAnomalies.detectAnomalies()** (line ~100): Query catalog mappings → if exists, query selling price → enrich anomaly with margin impact
4. **catalog-item-manager.tsx** (line ~380): Add `onClick` handler to navigate to `/sales-invoices/catalog/${item._id}`
