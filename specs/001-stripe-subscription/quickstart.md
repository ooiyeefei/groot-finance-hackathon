# Quickstart: Stripe Subscription Integration

**Feature**: 001-stripe-subscription
**Date**: 2025-12-27
**Pattern**: [Next.js SaaS Starter](https://github.com/nextjs/saas-starter)

This guide helps developers get started with the Stripe subscription integration for local development.

> **Design Principle**: Follow the Vercel SaaS Starter pattern - subscription fields on `businesses` table, Stripe Checkout + Customer Portal, simple webhook handlers.

## Prerequisites

- Node.js 18+ and npm
- Stripe CLI installed ([download](https://stripe.com/docs/stripe-cli))
- Access to FinanSEAL Stripe test account

## 1. Install Dependencies

```bash
# Install Stripe SDK packages
npm install stripe @stripe/stripe-js
```

## 2. Configure Environment Variables

Add to your `.env.local`:

```env
# Stripe Test Mode Keys
STRIPE_SECRET_KEY=sk_test_xxx
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_test_xxx

# Webhook Secret (generated by Stripe CLI in step 4)
STRIPE_WEBHOOK_SECRET=whsec_xxx

# Product Price IDs (create in Stripe Dashboard)
STRIPE_PRO_PRICE_ID=price_xxx
STRIPE_ENTERPRISE_PRICE_ID=price_xxx
```

### Getting Test Keys

1. Go to [Stripe Dashboard](https://dashboard.stripe.com/test/apikeys)
2. Ensure **Test mode** is enabled (toggle in top-right)
3. Copy "Publishable key" → `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY`
4. Copy "Secret key" → `STRIPE_SECRET_KEY`

## 3. Create Stripe Products

In the Stripe Dashboard (Test mode):

### Create Pro Plan
1. Go to Products → Add Product
2. Name: "FinanSEAL Pro"
3. Pricing: $XX/month (recurring)
4. Copy Price ID → `STRIPE_PRO_PRICE_ID`

### Create Enterprise Plan
1. Go to Products → Add Product
2. Name: "FinanSEAL Enterprise"
3. Pricing: $XX/month (recurring)
4. Copy Price ID → `STRIPE_ENTERPRISE_PRICE_ID`

## 4. Set Up Webhook Forwarding (Local Development)

The Stripe CLI forwards webhook events to your local server:

```bash
# Login to Stripe (one-time)
stripe login

# Forward webhooks to local endpoint
stripe listen --forward-to localhost:3000/api/v1/billing/webhooks
```

**Important:** Copy the webhook signing secret displayed (`whsec_xxx`) to `STRIPE_WEBHOOK_SECRET`.

Keep this terminal running during development.

## 5. Configure Customer Portal

In Stripe Dashboard → Settings → Billing → Customer portal:

1. Enable these features:
   - ✅ Update subscriptions (allow plan switching)
   - ✅ Cancel subscriptions (with feedback collection)
   - ✅ Update payment methods
   - ✅ View invoice history

2. Configure proration:
   - Set to "Always invoice immediately"

3. Save configuration

## 6. Run Database Migrations

```bash
# Apply billing tables migration
supabase db push

# Or if using migration files:
supabase migration up
```

## 7. Verify Setup

### Test Checkout Flow
```bash
# Trigger a test checkout session creation
curl -X POST http://localhost:3000/api/v1/billing/checkout \
  -H "Authorization: Bearer YOUR_CLERK_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"priceId": "price_xxx"}'
```

### Test Webhook Processing
```bash
# Trigger test webhook events via Stripe CLI
stripe trigger customer.subscription.created
stripe trigger invoice.payment_succeeded
stripe trigger customer.subscription.deleted
```

### Check Database
```sql
-- Verify subscription was created
SELECT * FROM subscriptions ORDER BY created_at DESC LIMIT 1;

-- Verify event was recorded (idempotency)
SELECT * FROM stripe_events ORDER BY created_at DESC LIMIT 5;
```

## Common Issues

### "No such price" Error
- Verify `STRIPE_PRO_PRICE_ID` matches your Stripe Dashboard
- Ensure you're using test mode keys with test mode prices

### Webhook Signature Verification Failed
- Restart `stripe listen` command
- Copy the new `whsec_xxx` to `.env.local`
- Restart Next.js dev server

### Customer Portal Not Working
- Ensure Customer Portal is configured in Stripe Dashboard
- Verify business has `stripe_customer_id` populated

### Webhooks Not Received
- Check `stripe listen` is running
- Check terminal for webhook delivery logs
- Verify endpoint path matches: `/api/v1/billing/webhooks`
